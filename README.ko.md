# EPUB Translator

> 🇺🇸 [View in English](./README.md)

EPUB 전자책의 구조와 서식을 완전히 보존하면서 문맥을 고려한 번역을 생성하는 AI 기반 번역 도구. 번역 결과를 검토하고 수정할 수 있는 웹 UI를 내장하고 있습니다.

## 배경

EPUB은 XML 기반 전자책 포맷으로, 콘텐츠와 마크업이 긴밀하게 뒤섞여 있습니다. 모든 단락에는 강조, 링크, 주석, 특수 문자 등 텍스트와 분리할 수 없는 인라인 요소들이 포함되어 있습니다.

이 때문에 EPUB 번역은 일반 텍스트 번역보다 훨씬 까다롭습니다. 원문을 그대로 LLM에 넣는 단순한 방식은 구조 손상의 위험이 있습니다 — 번역 자체는 올바르더라도 태그가 재작성·재배치·누락되어 렌더링된 전자책이 망가질 수 있습니다.

이 프로젝트의 핵심 목표는 **구조 손실 없이 EPUB을 번역**하는 것입니다. 원문의 모든 태그, 링크, 요소가 정확히 원래 위치에 유지됩니다.

---

## 기능 요약

- EPUB을 업로드하고 언어를 선택하면 완전히 번역된 EPUB을 다운로드할 수 있습니다
- 번역 결과물은 유효한 EPUB 파일입니다 — 챕터 구조, 서식, 링크가 그대로 유지됩니다
- 소설, 교재 등 장편 문서를 체크포인트 기반으로 재개 가능하게 처리합니다
- 진행 상황 모니터링, 결과 검토, 인라인 편집을 위한 웹 UI를 내장합니다

## 파이프라인

```
EPUB 입력
    │
    ▼
[Stage 1] 추출
    - EPUB spine을 순회하며 번역 대상 콘텐츠를 식별
    - LLM에는 마크업 없이 순수한 문장만 전달
    - 번역 불가 영역(코드, 수식, SVG 등)은 완전히 건너뜀
    │
    ▼
[Stage 2] 컨텍스트 구축
    - 번역 시작 전 전체 문서를 분석
    - 일관된 용어 번역을 위한 도메인 용어 사전 추출
    - 번역 맥락을 위한 섹션별 요약 생성
    - 섹션 간 용어 충돌은 빈도 기반으로 해결
    │
    ▼
[Stage 3] 번역
    - 용어 사전과 요약을 컨텍스트로 함께 전달하며 배치 번역
    - 구조화된 출력으로 입력 단락과 번역의 1:1 매핑 보장
    │
    ▼
[Stage 4] 삽입
    - 번역된 텍스트를 원래 마크업에 재결합
    - 각 챕터 문서를 원위치에서 패치
    - 유효한 EPUB 아카이브로 재패키징
```

## 기술적 도전 과제

### 1. 번역 과정에서 인라인 마크업 보존

EPUB 단락에는 링크, 강조, 주석 등 다양한 인라인 태그가 산문 안에 섞여 있습니다. LLM이 프롬프트에 마크업이 포함되면 이를 재작성하거나 재배치하는 경향이 있어, 번역 결과는 올바르더라도 문서 구조가 손상될 수 있습니다. LLM이 보는 것과 최종 문서에 들어가는 것을 깔끔하게 분리하고, 번역 후 두 가지를 신뢰성 있게 다시 합칠 수 있도록 설계했습니다.

### 2. 번역하지 말아야 할 것 식별

XHTML 문서의 모든 내용이 번역 대상은 아닙니다. 코드 블록, 수식, SVG, 서식화된 텍스트는 모든 하위 요소를 포함해 완전히 건드리지 않아야 합니다. 이러한 영역을 식별하고 중첩된 문서 트리 전체에 일관되게 "건너뛰기" 판단을 전파하는 필터링 로직을 구현했습니다.

### 3. 장편 문서 전반에서 번역 일관성 유지

장편 소설은 단일 컨텍스트 윈도우에 담기지 않습니다. 문서를 배치로 나누면 같은 용어가 섹션마다 다르게 번역되고, 각 배치는 이전 내용을 알 수 없습니다. 토큰 비용을 키우지 않으면서 모든 배치에 함께 전달되는 경량 컨텍스트 레이어(용어 사전과 요약)를 설계했습니다.

### 4. 장편 번역 작업의 중단 및 재개

소설 한 권을 번역하면 수백 번의 API 호출이 발생합니다. 네트워크 장애, 레이트 리밋, 수동 중지는 불가피하고, 매번 처음부터 다시 번역하는 것은 낭비입니다. 작업을 중복 없이 중간에서 재개할 만큼 세밀하면서도 상태를 작고 이식 가능하게 유지하는 체크포인팅 시스템을 구현했습니다.

### 5. 용어집 검토를 위한 Human-in-the-loop 워크플로우

전문 번역이나 정밀도가 중요한 경우, 자동 용어 추출만으로는 충분하지 않습니다 — 번역 시작 전에 사람이 도메인 용어를 검증해야 합니다. 표준 자동화 흐름을 복잡하게 만들지 않으면서 파이프라인을 중간에 멈추고, 추출된 용어를 검토용으로 제공하고, 수정을 받아 원활하게 재개하는 워크플로우를 설계했습니다.

## 워크플로우 모드

**Classic 모드**: 업로드 → 번역 → 다운로드. 완전 자동화.

**Glossary review 모드**: 컨텍스트 구축 단계 완료 후, 번역 시작 전에 파이프라인이 일시 중지됩니다. 추출된 용어 사전이 웹 UI에 표시되어 사람이 검토하고 수정할 수 있습니다. 승인 후 번역이 재개되면 수정된 용어 사전이 모든 번역 호출에 적용됩니다.

## 아키텍처

```
┌──────────────────────────────────────────┐
│              데스크탑 앱                  │
│   pywebview 윈도우 (브라우저 창 없음)     │
└─────────────────┬────────────────────────┘
                  │  HTTP (localhost)
┌─────────────────▼────────────────────────┐
│           FastAPI 백엔드                  │
│                                          │
│  작업 관리, SSE 진행률 스트리밍           │
│  용어집 검토 API, 인라인 편집 API         │
│  다운로드 엔드포인트                      │
└─────────────────┬────────────────────────┘
                  │
┌─────────────────▼────────────────────────┐
│             번역 파이프라인               │
│                                          │
│  추출 → 전처리 → 번역 → 삽입             │
│                                          │
│  체크포인트 기반 — 중단 시 재개 가능      │
└──────────────────────────────────────────┘
```

프론트엔드는 React + TypeScript SPA로, FastAPI가 정적 파일로 서빙합니다. 데스크탑 셸은 전체를 pywebview 윈도우로 감싸 네이티브 앱처럼 동작합니다.

## 웹 UI

**메인 페이지** — EPUB 업로드, SSE 기반 실시간 진행 모니터링, 설정 관리.

**용어집 검토 페이지** — 번역 시작 전 추출된 전체 용어를 확인하고 수정할 수 있습니다 (glossary review 모드 전용).

**결과 뷰어** — 원문과 번역문 나란히 보기. 번역 단락 인라인 편집을 지원하며, 변경 내용이 EPUB에 저장됩니다.
